# Family Budget Bot

Телеграм-бот для семейного чата, который **следит за тратами и раз в неделю сам
присылает разбор**: на что и по каким категориям ушли деньги. Работает на
**YandexGPT**.

Вы просто пишете в общий чат, что купили и почём. Бот разбирает это, копит в
базе, а в понедельник утром выкладывает сводку за прошлую неделю. Плюс отвечает
на вопросы о расходах и — по запросу — читает чеки и подсказывает рецепты.

```
Жена:  молоко 89, батон 45, куриное филе 1.2кг 420
Бот:   ✅ Записал, 554 ₽
       • молоко (1 л) — 89 ₽  молочное и яйца
       • батон — 45 ₽  хлеб и выпечка
       • куриное филе (1.2 кг) — 420 ₽  мясо и рыба

           …спустя неделю, в понедельник утром, сам…

Бот:   Расходы за прошлую неделю
       Всего: 7 240 ₽

       • мясо и рыба — 2 480 ₽  (34%, 6 поз.)
       • овощи и фрукты — 1 510 ₽  (21%, 14 поз.)
       • молочное и яйца — 980 ₽  (14%, 9 поз.)
       …

       Что видно
       Больше всего ушло на мясо и рыбу — на 40% больше, чем неделей раньше.
       Заметный разовый расход: сыр за 890 ₽. В остальном траты как обычно.
```

## Как это работает

Главное — сбор трат и их анализ. Всё остальное второстепенно и включается только
по команде.

```
        сообщение в чате
               │
               ├─ команда (/digest, /stats, /receipt …)  ──►  ответ из базы
               │
               ├─ обращение к боту                 ──────────►  YandexGPT + контекст
               │  (реплай / «бот, …» / @упоминание)             из базы  →  ответ
               │
               ├─ в тексте нет цифр                ──────────►  игнорируем
               │                                                (экономим деньги)
               │
               └─ есть цифры  ──►  YandexGPT разбирает текст
                                          │
                          не покупка ──────┴────── покупка
                              │                       │
                           молчим                пишем в SQLite
                                                 + подтверждение

        раз в неделю по расписанию
               │
               └─►  сводка за прошлую неделю + анализ  ──►  во все активные чаты
```

Ключевые решения:

**Данные собираются сами.** Пока вы пишете покупки в чат, бот тихо копит их в
базе. Ничего специально запускать не надо — статистика набирается фоном.

**Фото и рецепты — по запросу.** Бот не лезет в каждую присланную картинку и не
навязывает советы. Чек разберёт по команде `/receipt`, рецепт предложит по
`/recipe`. Это сделано осознанно: главная функция — деньги, остальное не должно
шуметь.

**Разбор через строгую схему.** Модель получает JSON Schema (генерируется из
`models.py`) и возвращает структуру, а не текст, который пришлось бы разбирать
регулярками. Категорию модель тоже выбирает строго из списка.

**Фильтр перед платным вызовом.** Сообщения без цифр в API не уходят вообще —
а таких в семейном чате большинство. Это экономит основную часть денег.

**Категории фиксированы.** Список в `models.py`. Если разрешить модели придумывать
категории на лету, статистика превратится в кашу из синонимов («продукты»,
«еда», «продукты питания»). Хотите свои — правьте `Category` в `models.py`.

**В модель уходит не весь чат.** Только выжимка: агрегаты по категориям и
последние N распознанных покупок (`CONTEXT_PURCHASES_LIMIT`). Переписка как
таковая никуда не отправляется.

## Еженедельный дайджест

Раз в неделю (по умолчанию — понедельник, 10:00) бот сам публикует в каждый
активный чат разбор трат за **прошлую календарную неделю** (пн–вс):

* таблицу по категориям с долями и числом позиций;
* самые дорогие покупки;
* короткий анализ от нейросети: на что ушло больше всего, что изменилось
  по сравнению с позапрошлой неделей, было ли что-то необычное.

Настраивается в `.env`:

```ini
WEEKLY_DIGEST=true            # выключить рассылку целиком — false
WEEKLY_DIGEST_WEEKDAY=0       # 0 — понедельник … 6 — воскресенье
WEEKLY_DIGEST_TIME=10:00      # время по TIMEZONE
WEEKLY_DIGEST_ANALYSIS=true   # false — только цифры, без вызова модели
```

**Кому приходит.** Всем чатам, где записана хоть одна покупка. Если задан
`ALLOWED_CHAT_IDS`, то только чатам из белого списка.

**Если за неделю ничего не куплено** — дайджест не публикуется (еженедельное
«покупок не записано» быстро надоедает).

**Посмотреть в любой момент** — команда `/digest`: это тот же разбор, что уходит
по расписанию, но по запросу.

> Планировщик простой: бот спит до времени публикации и просыпается. Если в этот
> момент бот был выключен, неделя пропускается — следующая придёт как обычно.
> Пропущенное задним числом не досылается.

## Быстрый старт

Нужны: Python 3.11+ (проверено на 3.14), аккаунт Telegram и доступ к Yandex Cloud.

### 1. Создать бота в Telegram

1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot` → придумайте имя и username.
2. Скопируйте токен вида `1234567890:AAH...` — это `TELEGRAM_BOT_TOKEN`.

> #### ⚠️ Обязательно: выключите privacy mode
>
> По умолчанию бот в группе видит **только** команды со слешем — обычные
> сообщения о покупках до него не долетят, и статистика не наберётся.
>
> В @BotFather: `/mybots` → выберите бота → **Bot Settings** → **Group Privacy**
> → **Turn off**. Должно быть написано «Privacy mode is disabled».
>
> Если бот уже был добавлен в группу — **удалите его из группы и добавьте
> заново**, иначе настройка не применится.

### 2. Получить доступ к YandexGPT

1. В [консоли Yandex Cloud](https://console.yandex.cloud/) создайте сервисный
   аккаунт и выдайте ему роль `ai.languageModels.user`. Если планируете
   разбирать чеки командой `/receipt` — добавьте ещё `ai.vision.user`.
2. Сервисный аккаунт → **Создать новый ключ** → **API-ключ**. Это `YANDEX_API_KEY`.
3. `YANDEX_FOLDER_ID` — идентификатор каталога, он же в адресной строке консоли.
4. К облаку должен быть привязан платёжный аккаунт. Тарификация — за токены,
   см. [раздел про стоимость](#сколько-это-стоит).

### 3. Поставить зависимости

```bash
cd ~/PycharmProjects/family-budget-bot
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

В PyCharm: **Settings → Project → Python Interpreter → Add → Existing** →
укажите `.venv/bin/python`.

### 4. Заполнить .env

```bash
cp .env.example .env
```

Впишите `TELEGRAM_BOT_TOKEN`, `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`.
`ALLOWED_CHAT_IDS` пока оставьте пустым — узнаем на следующем шаге.

### 5. Первый запуск

```bash
python bot.py
```

В логе должно появиться `Запущен как @ваш_бот`. Теперь:

1. Добавьте бота в вашу семейную группу.
2. Отправьте в группе `/chatid` — бот ответит ID (у групп он отрицательный,
   например `-1001234567890`).
3. Впишите этот ID в `ALLOWED_CHAT_IDS` в `.env`.
4. Перезапустите бота (`Ctrl+C`, затем `python bot.py`).

Готово. Напишите в чат `молоко 89, хлеб 45` — бот должен ответить. А в понедельник
утром придёт первый дайджест (если к тому времени накопятся покупки).

> Пока `ALLOWED_CHAT_IDS` пуст, бот отвечает в **любом** чате, куда его добавят,
> и тратит ваши деньги на API. Заполните его.

## Команды

**Статистика (главное)**

| Команда | Что делает |
|---|---|
| `/digest` | Разбор трат за прошлую неделю — то же, что приходит по расписанию |
| `/stats` | Расходы по категориям за текущий месяц |
| `/stats неделя` | То же за 7 дней. Ещё: `сегодня`, `вчера`, `месяц`, `год`, `всё`, число дней (`/stats 30`) |
| `/ask <вопрос>` | Вопрос по покупкам: `/ask на что ушло больше всего в июне?` |
| `/cost` | Сколько потрачено на YandexGPT. Периоды те же, что у `/stats` |

**Чеки и продукты (по запросу)**

| Команда | Что делает |
|---|---|
| `/receipt` | Разобрать фото чека (подпись к фото или ответ на фото) |
| `/fridge` | Что, скорее всего, лежит дома |
| `/recipe` | Что приготовить из этого. Можно с пожеланием: `/recipe быстро и без мяса` |
| `/ate <продукт>` | Отметить съеденным, чтобы не мозолил глаза в `/fridge` |

**Прочее**

| Команда | Что делает |
|---|---|
| `/undo` | Удалить последнюю записанную покупку (если бот разобрал неправильно) |
| `/export` | Выгрузить все покупки в CSV (открывается в Excel) |
| `/categories` | Список категорий |
| `/chatid` | ID текущего чата |
| `/help` | Справка |

**Спросить без команды** — ответьте реплаем на сообщение бота, начните со слова
«бот» или упомяните `@username`:

```
бот, сколько мы тратим на кафе в среднем за месяц?
бот, на что в этом месяце ушло больше всего?
```

## Чеки и продукты

Это второстепенные функции: они не работают сами, только по команде.

### Чеки — `/receipt`

Бот не трогает картинки, которые просто прислали в чат. Чтобы разобрать чек,
вызовите разбор явно, любым из двух способов:

* отправьте **фото с подписью** `/receipt` (или `/чек`);
* ответьте `/receipt` на уже отправленное фото.

Дальше бот скачает картинку, подготовит её (`images.py`) и прочитает:

* **разворачивает по EXIF** — снимок, сделанный вертикально, иначе уедет набок;
* **ужимает до 2576 px** по длинной стороне — платить за лишние пиксели незачем;
* приводит к JPEG (принимаются и PNG, и HEIC, и картинка, отправленная файлом).

**Как читается чек.** Текстовая модель YandexGPT картинок не видит, поэтому фото
сначала проходит через **Yandex Vision OCR**, а модель разбирает уже
распознанный текст (`prompts.py` → `RECEIPT_TEXT_SYSTEM`) — она знает, что строки
могли перепутаться, а символы прочитаться неверно. Получается два вызова на один
чек, и в `/cost` они попадут двумя записями: распознавание тарифицируется за
страницу, разбор — за токены.

> Для OCR сервисному аккаунту нужна роль `ai.vision.user` — отдельно от
> `ai.languageModels.user`.

Что понимает в самом чеке:

* отсекает служебные строки: ИТОГО, НДС, СДАЧА, кассир, ФН/ФД/ФП, ИНН, реклама;
* весовой товар `0.784 x 289.00 = 226.58` → 0.784 кг за 226,58 ₽;
* строку «СКИДКА ПО КАРТЕ» вычитает из позиции, а не заводит отдельной покупкой;
* сокращения расшифровывает: `П/Ф КУР ФИЛЕ ОХЛ` → «куриное филе»;
* берёт из шапки магазин и дату — покупка запишется числом чека.

**Сверка с ИТОГО.** Бот сравнивает сумму распознанных позиций со строкой ИТОГО.
Расходятся больше чем на рубль — предупредит:

```
⚠️ В чеке ИТОГО 1 552,12 ₽, а по позициям 1 490,00 ₽. Проверьте, при ошибке — /undo
```

**Полезно знать:**

* **Длинный чек лучше отправлять файлом**, а не фото — Telegram сжимает фотографии
  до 1280 px, и мелкий шрифт может стать нечитаемым.
* Ошибся — `/undo` удалит запись целиком.
* Выключить команду совсем: `READ_RECEIPTS=false` в `.env`.

### Продукты — `/fridge` и `/recipe`

Бот помечает съедобные покупки и на их основе показывает, что, скорее всего,
лежит дома (`/fridge`), и предлагает блюда (`/recipe`). «Холодильник» — это
эвристика: продукты, купленные за последние `FRIDGE_WINDOW_DAYS` дней и не
отмеченные командой `/ate`. Бот не знает, что вы реально съели.

## Сколько это стоит

Платятся только вызовы YandexGPT — Telegram и SQLite бесплатны. Сообщения без
цифр в API не уходят совсем. Виды вызовов:

| Что | Когда | Какая модель |
|---|---|---|
| Разбор текста о покупке | на каждое сообщение с цифрами | `YANDEX_PARSER_MODEL` |
| Анализ недели | раз в неделю | `YANDEX_MODEL` |
| Вопрос по тратам | по обращению / `/ask` | `YANDEX_MODEL` |
| Фото чека | по команде `/receipt` | OCR (за страницу) + `YANDEX_VISION_MODEL` |
| Рецепт | по команде `/recipe` | `YANDEX_MODEL` |

Основной поток — разбор коротких сообщений о покупках, поэтому под него по
умолчанию стоит дешёвая `yandexgpt-lite`. Порядок величин для семьи из двух
человек — единицы–десятки рублей в месяц.

### Считайте по факту, а не по прикидкам

Бот ведёт собственный учёт: после каждого обращения он берёт из ответа число
токенов, считает стоимость по прайсу из `.env` и складывает в таблицу `usage_log`.
Команда `/cost`:

```
Расходы на нейросеть за июль
Всего: 84 ₽ за 320 вызовов

По операциям
• чтение чеков — 42 ₽ (10)
• разбор покупок — 24 ₽ (300)
• вопросы — 12 ₽ (6)
• анализ недели — 6 ₽ (4)

По моделям
• yandexgpt/latest — 60 ₽ (20)
• yandexgpt-lite/latest — 24 ₽ (300)

Токенов: 480K вход / 96K выход
При таком темпе за месяц выйдет ~112 ₽
```

Периоды те же, что у `/stats`: `/cost неделя`, `/cost сегодня`, `/cost всё`.
Прогноз на месяц показывается только когда смотрите текущий месяц целиком.

**Про точность прайса.** Цена Яндекса зависит от тарифа и объёма. В `.env` стоят
ориентировочные ставки (`YANDEX_PRICE_*`) — впишите свои реальные, иначе `/cost`
покажет оценку. Стоимость считается в момент вызова и хранится посчитанной:
цены со временем меняются, а пересчитывать прошлое по новому прайсу неверно.

**Как снизить расходы:**

1. Разбор уже на `yandexgpt-lite` — это самый частый вызов. Старшую модель
   оставьте на вопросы и анализ, где она заметно лучше.
2. **Писать покупки одним сообщением**, а не пятью подряд — один вызов вместо пяти.
3. Выключить анализ недели (`WEEKLY_DIGEST_ANALYSIS=false`) — цифры останутся,
   вызов модели уйдёт.
4. Следите за расходом в консоли Yandex Cloud, там же можно поставить лимит.

Что где менять:

* **бот неверно разбирает покупки** → `prompts.py`, `PARSER_SYSTEM`;
* **бот неверно читает чеки** → `prompts.py`, `RECEIPT_TEXT_SYSTEM`;
* **не нравится тон анализа/ответов** → `prompts.py`, `WEEKLY_SYSTEM` / `ASSISTANT_SYSTEM`;
* **время и содержимое дайджеста** → `.env` (`WEEKLY_DIGEST_*`) и `services.weekly_digest`;
* **вид отчётов** → `services.py`;
* **новая команда** → `handlers/commands.py`.

## Приватность и безопасность

* `.env` и папка `data/` — в `.gitignore`. **Перед первым `git push` проверьте,
  что там нет токенов:** `git status` и `git log -p | grep -iE 'AQVN|t1\.'`.
* **Куда уходят данные.** Тексты сообщений с цифрами (для разбора), выжимка из
  базы (для ответов и анализа) и фотографии чеков отправляются в Yandex Cloud.
  Переписка целиком не отправляется — только то, что бот распознал как покупку.
* Запросам в Yandex Vision OCR бот проставляет `x-data-logging-enabled: false` —
  чеки не должны оседать в логах Яндекса.
* База лежит локально в `data/bot.sqlite3`. Это обычный SQLite-файл без шифрования —
  на сервере позаботьтесь о правах доступа (`chmod 600`).
* `ALLOWED_CHAT_IDS` — единственное, что мешает постороннему добавить вашего бота
  к себе и тратить ваш баланс API. Заполните его.
* Если токен утёк: @BotFather → `/mybots` → **API Token** → **Revoke**.
  Ключ Yandex Cloud — удалите в консоли и создайте новый.

## Если что-то не работает

**Бот молчит на обычные сообщения, но отвечает на команды.**
Не выключен privacy mode. См. [шаг 1](#1-создать-бота-в-telegram) — и не забудьте
после этого удалить бота из группы и добавить заново.

**Бот вообще не отвечает в группе.**
Проверьте лог. Если там `Сообщение из чужого чата -100…` — этот ID не вписан в
`ALLOWED_CHAT_IDS`.

**`Не задана обязательная переменная …`**
Нет файла `.env` или в нём пустое значение. `cp .env.example .env` и заполните.

**`Yandex Cloud не принял ключ`.**
Проверьте `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`, а главное — роли сервисного
аккаунта: нужна `ai.languageModels.user`, а для чтения чеков ещё `ai.vision.user`.

**`На платёжном аккаунте Yandex Cloud закончились деньги`.**
Пополните счёт в консоли Yandex Cloud. Пока баланс нулевой, бот не сможет
записывать покупки и отвечать — он предупредит об этом в чате (не чаще раза в
10 минут, чтобы не спамить).

**Дайджест не приходит.**
Проверьте: `WEEKLY_DIGEST=true`; в чате есть записанные за прошлую неделю
покупки (пустую неделю бот пропускает); бот был запущен в момент публикации
(пропущенное не досылается). В логе при старте видно `Следующий дайджест: …`.

**`/receipt` отвечает «Разбор чеков выключен».**
`READ_RECEIPTS=false` в `.env`. Поставьте `true`.

**Чек не разобрался / не сошлось ИТОГО.**
`/undo` удалит запись. Переснимите чек ровнее и при хорошем свете либо впишите
позиции текстом. Систематически плохо читается магазин — добавьте пример его
формата в `RECEIPT_TEXT_SYSTEM` в `prompts.py`.

**Бот записал покупку неправильно.**
`/undo` удалит последнюю запись. Если ошибается систематически — поправьте
`PARSER_SYSTEM` в `prompts.py`.

**`TelegramConflictError: terminated by other getUpdates request`.**
Запущено два экземпляра бота с одним токеном. Остановите лишний.

### v3 — вынос на сервер
Сейчас бот работает через long polling — это значит, что он живёт ровно столько,
сколько открыт терминал на ноутбуке. Для круглосуточной работы (и чтобы дайджест
приходил, даже когда ноутбук закрыт) нужен сервер.

**Шаг 1. Арендовать VPS.** Хватит самого дешёвого: 1 CPU / 1 GB RAM
(Timeweb, Selectel, Hetzner — от ~200 ₽/мес). ОС — Ubuntu 24.04 LTS.

**Шаг 2. Развернуть.**
```bash
ssh root@ваш-сервер
apt update && apt install -y python3-venv git
git clone https://github.com/ВАШ_ЛОГИН/family-budget-bot.git /opt/family-budget-bot
cd /opt/family-budget-bot
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cp .env.example .env && nano .env      # вписать токены
chmod 600 .env
```

**Шаг 3. Сделать systemd-сервис**, чтобы бот стартовал сам и переживал
перезагрузки — `/etc/systemd/system/family-budget-bot.service`:
```ini
[Unit]
Description=Family Budget Bot
After=network-online.target

[Service]
WorkingDirectory=/opt/family-budget-bot
ExecStart=/opt/family-budget-bot/.venv/bin/python bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```
```bash
systemctl daemon-reload
systemctl enable --now family-budget-bot
journalctl -u family-budget-bot -f     # смотреть логи
```

**Шаг 4. Бэкапы базы.** Cron раз в сутки:
```
0 4 * * * sqlite3 /opt/family-budget-bot/data/bot.sqlite3 ".backup /root/backup/bot-$(date +\%F).sqlite3"
```
