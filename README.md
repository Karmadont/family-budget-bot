# Family Budget Bot

Телеграм-бот для семейного чата, куда вы скидываете, что и почём купили. Бот
разбирает эти сообщения нейросетью — на выбор **Claude**, **YandexGPT** или
**GigaChat**, переключается одной строкой в `.env`, — складывает в базу и умеет:

* вести учёт покупок по категориям и считать статистику расходов;
* **читать фотографии кассовых чеков** и разносить их по позициям;
* подсказывать, что приготовить из того, что сейчас лежит в холодильнике;
* отвечать на свободные вопросы про ваши траты и продукты.

Никаких форм и кнопок: пишете в чат как обычно или кидаете фото чека,
разбором занимается нейросеть.

```
Жена:  молоко 89, батон 45, куриное филе 1.2кг 420
Бот:   ✅ Записал, 554 ₽
       • молоко (1 л) — 89 ₽  молочное и яйца
       • батон — 45 ₽  хлеб и выпечка
       • куриное филе (1.2 кг) — 420 ₽  мясо и рыба

Вы:    [фото чека из Пятёрочки]
Бот:   🧾 Читаю чек…
       ✅ Записал 7 поз. на 1 552,12 ₽ · Пятёрочка
       • молоко — 89,90 ₽  молочное и яйца
       • хлеб бородинский (2 шт) — 91 ₽  хлеб и выпечка
       • куриное филе (1.184 кг) — 413,22 ₽  мясо и рыба
       …

Вы:    бот, что приготовить на ужин?
Бот:   Куриное филе куплено 3 дня назад — его в первую очередь. …
```

## Как это работает

Бот сидит в вашей группе и читает все сообщения. Для каждого он решает, что делать:

```
        сообщение в чате
               │
               ├─ это команда (/stats, /fridge …)  ──────────►  ответ из базы
               │
               ├─ обращение к боту                 ──────────►  нейросеть + контекст
               │  (реплай / «бот, …» / @упоминание)             из базы  →  ответ
               │
               ├─ фотография ──►  скачиваем, поворачиваем по EXIF,
               │                  ужимаем до 2576 px
               │                          │
               │                          ▼
               │                  разбор чека ──►  позиции в SQLite
               │
               ├─ в тексте нет цифр                ──────────►  игнорируем
               │                                                (экономим деньги)
               │
               └─ есть цифры  ──►  нейросеть разбирает текст
                                          │
                          не покупка ──────┴────── покупка
                              │                       │
                           молчим                пишем в SQLite
                                                 + подтверждение
```

Ключевые решения:

**Провайдер — деталь реализации.** Логика бота не знает, кто именно отвечает.
Всё общение с моделью идёт через `llm.py`, а конкретные реализации лежат в
`providers/`. Добавить четвёртого — это один новый файл на ~150 строк.

**Разбор через строгую схему.** Модель получает JSON Schema (генерируется из
`models.py`) и не может вернуть невалидный ответ или выдуманную категорию.
Механизм у всех троих разный — у Claude это `output_config.format`, у
YandexGPT `json_schema`, у GigaChat вызов функции, — но снаружи это одно и то же.

**Фильтр перед платным вызовом.** Сообщения без цифр в API не уходят вообще —
а таких в семейном чате большинство. Это экономит основную часть денег.

**Категории фиксированы.** Список в `models.py`. Если разрешить модели придумывать
категории на лету, статистика превратится в кашу из синонимов («продукты»,
«еда», «продукты питания»). Хотите свои — правьте `Category` в `models.py`,
база подхватит автоматически.

**В модель уходит не весь чат.** Только выжимка: агрегаты по категориям за месяц,
список продуктов «в холодильнике» и последние N распознанных покупок
(`CONTEXT_PURCHASES_LIMIT`). Переписка как таковая никуда не отправляется.

**«Холодильник» — это эвристика.** Бот не знает, что вы съели. Он показывает еду,
купленную за последние `FRIDGE_WINDOW_DAYS` дней и не отмеченную командой `/ate`.
Для «что приготовить» этого достаточно.

## Выбор нейросети

Кто работает — решают две строки в `.env`:

```ini
LLM_PROVIDER=claude        # claude | yandexgpt | gigachat — текст и вопросы
VISION_PROVIDER=           # кто читает чеки. Пусто = тот же
```

Ключи нужны только от включённых провайдеров: с `LLM_PROVIDER=gigachat`
переменную `ANTHROPIC_API_KEY` можно вообще не заполнять. Кто сейчас работает —
покажет команда `/provider`.

| | Claude | YandexGPT | GigaChat |
|---|---|---|---|
| Ключ | `ANTHROPIC_API_KEY` | `YANDEX_API_KEY` + `YANDEX_FOLDER_ID` | `GIGACHAT_AUTH_KEY` |
| Строгий JSON | `output_config.format` | `json_schema` | вызов функции |
| Фото чека | видит картинку | **только через OCR** | видит картинку (кроме Lite) |
| Валюта | доллары | рубли | рубли |
| Оплата из РФ | сложно | обычная | обычная |

**Зачем `VISION_PROVIDER` отдельно.** Зрение — самое слабое место в этой связке.
Можно гонять весь чат через GigaChat, а чеки оставить на Claude:

```ini
LLM_PROVIDER=gigachat
VISION_PROVIDER=claude
```

`/cost` покажет расходы обоих — по этому и сравнивайте.

**Как честно сравнить.** Поработайте неделю на одном провайдере, посмотрите
`/cost неделя`, переключите и сравните через неделю. База общая, история
расходов сохраняется по каждому провайдеру отдельно — старые записи никуда
не денутся и не смешаются.

> Учтите: у Anthropic цена в долларах, у Яндекса и Сбера — в рублях. Чтобы
> `/cost` свёл их в одну цифру, задайте примерный курс в `USD_RATE`.

## Фото чеков

Просто отправьте фотографию чека в чат — с подписью или без. Бот ответит
«🧾 Читаю чек…» и через несколько секунд заменит это сообщение на разбор.

**Два пути в зависимости от провайдера.** Claude и GigaChat смотрят на картинку
сами. Текстовый YandexGPT картинок не видит, поэтому фото сначала уходит в
**Yandex Vision OCR**, а модели достаётся распознанный текст — и разбирает его
она по отдельному промпту (`RECEIPT_TEXT_SYSTEM`), который знает, что строки
могли перепутаться, а символы прочитаться неверно. Вызова получается два, и в
`/cost` они попадут двумя записями: распознавание тарифицируется за страницу,
а не за токены.

> Для OCR сервисному аккаунту Yandex Cloud нужна роль `ai.vision.user` —
> отдельно от `ai.languageModels.user`.

Что бот делает с картинкой перед отправкой в модель (`images.py`):

* **разворачивает по EXIF** — снимок, сделанный вертикально, иначе уедет набок,
  и модель будет читать чек боком;
* **ужимает до 2576 px** по длинной стороне — больше не использует ни одна из
  моделей, платить за лишние пиксели незачем;
* приводит к JPEG (принимаются и PNG, и HEIC, и картинка, отправленная файлом).

Что понимает в самом чеке (`prompts.py` → `RECEIPT_SYSTEM`):

* отсекает служебные строки: ИТОГО, НДС, СДАЧА, кассир, ФН/ФД/ФП, ИНН, реклама;
* весовой товар `0.784 x 289.00 = 226.58` → 0.784 кг за 226,58 ₽;
* строку «СКИДКА ПО КАРТЕ» вычитает из соответствующей позиции, а не заводит
  отдельной покупкой;
* сокращения расшифровывает: `П/Ф КУР ФИЛЕ ОХЛ` → «куриное филе»;
* берёт из шапки магазин и дату — покупка запишется числом чека, а не сегодняшним.

**Сверка с ИТОГО.** Бот отдельно читает строку ИТОГО и сравнивает её с суммой
распознанных позиций. Расходятся больше чем на рубль — предупредит:

```
⚠️ В чеке ИТОГО 1 552,12 ₽, а по позициям 1 490,00 ₽. Проверьте, при ошибке — /undo
```

Это главная защита от тихой ошибки распознавания: цифру бот мог прочесть неверно,
но не сойтись с ИТОГО он не может незаметно.

**Полезно знать:**

* **Длинный чек лучше отправлять файлом**, а не фото. Telegram сжимает фотографии
  до 1280 px, и мелкий шрифт внизу длинной ленты может стать нечитаемым.
  Файлом уходит оригинал, а до нужного размера бот ужмёт его сам.
* Если чек не влез в один кадр и вы шлёте несколько фото — бот разберёт **каждое
  отдельно**. Позиции сложатся правильно, но строка ИТОГО есть только на последнем
  снимке, поэтому по остальным будет предупреждение о расхождении. Это нормально.
* **Не дублируйте.** Если отправили фото чека, не пишите потом сумму текстом —
  бот запишет её второй раз. В подписи к фото сумму указывать можно: она идёт
  только на сверку.
* Фото не чека бот просто прокомментирует «не увидел чек» и ничего не запишет.
* Выключить целиком: `READ_RECEIPTS=false` в `.env`.

## Быстрый старт

Нужны: Python 3.11+ (проверено на 3.14), аккаунт Telegram и ключ **одного** из
трёх провайдеров — Anthropic, Yandex Cloud или GigaChat.

### 1. Создать бота в Telegram

1. Напишите [@BotFather](https://t.me/BotFather) → `/newbot` → придумайте имя и username.
2. Скопируйте токен вида `1234567890:AAH...` — это `TELEGRAM_BOT_TOKEN`.

> #### ⚠️ Обязательно: выключите privacy mode
>
> По умолчанию бот в группе видит **только** команды со слешем — обычные
> сообщения о покупках до него не долетят, и он будет молчать.
>
> В @BotFather: `/mybots` → выберите бота → **Bot Settings** → **Group Privacy**
> → **Turn off**. Должно быть написано «Privacy mode is disabled».
>
> Если бот уже был добавлен в группу — **удалите его из группы и добавьте
> заново**, иначе настройка не применится.

### 2. Получить ключ нейросети

Нужен ключ **одного** провайдера — того, который поставите в `LLM_PROVIDER`.

<details open>
<summary><b>Claude</b> (<code>LLM_PROVIDER=claude</code>)</summary>

1. [console.anthropic.com](https://console.anthropic.com/) → **API keys** → **Create key**.
2. Ключ вида `sk-ant-...` — это `ANTHROPIC_API_KEY`. Показывается один раз, сохраните.
3. Пополните баланс в **Billing** — без него API вернёт ошибку.

Это платный доступ к API, он **не входит** в подписку Claude Pro/Max и не связан
с ней.
</details>

<details>
<summary><b>YandexGPT</b> (<code>LLM_PROVIDER=yandexgpt</code>)</summary>

1. В [консоли Yandex Cloud](https://console.yandex.cloud/) создайте сервисный
   аккаунт и выдайте ему роль `ai.languageModels.user`. Если этот же провайдер
   будет читать чеки — добавьте `ai.vision.user`.
2. Сервисный аккаунт → **Создать новый ключ** → **API-ключ**. Это `YANDEX_API_KEY`.
3. `YANDEX_FOLDER_ID` — идентификатор каталога, он же в адресной строке консоли.
4. К облаку должен быть привязан платёжный аккаунт.
</details>

<details>
<summary><b>GigaChat</b> (<code>LLM_PROVIDER=gigachat</code>)</summary>

1. [developers.sber.ru/portal/products/gigachat-api](https://developers.sber.ru/portal/products/gigachat-api)
   → создайте проект (для дома подойдёт «для физических лиц»).
2. Скопируйте **ключ авторизации** — длинную строку base64. Это `GIGACHAT_AUTH_KEY`.
3. `GIGACHAT_SCOPE` — `GIGACHAT_API_PERS` для физлиц, `GIGACHAT_API_B2B` или
   `GIGACHAT_API_CORP` для юрлиц.
4. **Сертификат.** GigaChat подписан НУЦ Минцифры, и без его корневого
   сертификата соединение не установится. Скачайте с
   [gosuslugi.ru/crt](https://www.gosuslugi.ru/crt) и укажите путь к `.pem`
   в `GIGACHAT_CA_BUNDLE`. Крайний вариант — `GIGACHAT_VERIFY_SSL=false`,
   но тогда TLS-соединение никто не проверяет.
</details>

Тарификация везде за токены, см. [раздел про стоимость](#сколько-это-стоит).

### 3. Поставить зависимости

```bash
cd ~/PycharmProjects/family-budget-bot
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

В PyCharm: **Settings → Project → Python Interpreter → Add → Existing** →
укажите `.venv/bin/python`.

### 4. Заполнить .env

```bash
cp .env.example .env
```

Откройте `.env` и впишите `TELEGRAM_BOT_TOKEN`, выберите `LLM_PROVIDER` и
заполните ключи **только для него** — остальные оставьте пустыми.
`ALLOWED_CHAT_IDS` пока тоже пустой, узнаем на следующем шаге.

### 5. Первый запуск

```bash
python bot.py
```

В логе должно появиться `Запущен как @ваш_бот`. Теперь:

1. Добавьте бота в вашу семейную группу.
2. Отправьте в группе `/chatid` — бот ответит ID (у групп он отрицательный,
   например `-1001234567890`).
3. Впишите этот ID в `ALLOWED_CHAT_IDS` в `.env`.
4. Перезапустите бота (`Ctrl+C`, затем `python bot.py`).

Готово. Напишите в чат `молоко 89, хлеб 45` — бот должен ответить.

> Пока `ALLOWED_CHAT_IDS` пуст, бот отвечает в **любом** чате, куда его добавят,
> и тратит ваши деньги на API. Заполните его.

## Команды

| Команда | Что делает |
|---|---|
| `/stats` | Расходы по категориям за текущий месяц |
| `/stats неделя` | То же за 7 дней. Ещё: `сегодня`, `вчера`, `месяц`, `год`, `всё`, или число дней (`/stats 30`) |
| `/fridge` | Что, скорее всего, лежит дома |
| `/recipe` | Что приготовить из этого. Можно с пожеланием: `/recipe быстро и без мяса` |
| `/ask <вопрос>` | Вопрос по покупкам: `/ask на что ушло больше всего в июне?` |
| `/ate <продукт>` | Отметить съеденным, чтобы не мозолил глаза в `/fridge` |
| `/cost` | Сколько потрачено на нейросеть, с разбивкой по провайдерам. Периоды те же, что у `/stats` |
| `/undo` | Удалить последнюю записанную покупку (если бот разобрал неправильно) |
| `/export` | Выгрузить все покупки в CSV (открывается в Excel) |
| `/categories` | Список категорий |
| `/provider` | Какая нейросеть сейчас работает и на каких моделях |
| `/chatid` | ID текущего чата |
| `/help` | Справка |

**Спросить без команды** — ответьте реплаем на сообщение бота, начните со слова
«бот» или упомяните `@username`:

```
бот, сколько мы тратим на кафе в среднем за месяц?
бот, что можно приготовить, чтобы не пропало мясо?
```

## Настройки (.env)

**Общие**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | — | **Обязательно.** Токен от @BotFather |
| `ALLOWED_CHAT_IDS` | пусто | Белый список чатов через запятую. Пусто = любой чат |
| `LLM_PROVIDER` | `claude` | Кто разбирает текст и отвечает: `claude`, `yandexgpt`, `gigachat` |
| `VISION_PROVIDER` | = `LLM_PROVIDER` | Кто читает фото чеков |
| `READ_RECEIPTS` | `true` | Читать ли фотографии чеков |
| `MAX_IMAGE_MB` | `20` | Предел размера картинки (потолок Telegram — 20 МБ) |
| `LLM_TIMEOUT` | `120` | Таймаут запроса к YandexGPT/GigaChat, секунд |

**Claude** — нужны, если он стоит в `LLM_PROVIDER` или `VISION_PROVIDER`

| Переменная | По умолчанию | Описание |
|---|---|---|
| `ANTHROPIC_API_KEY` | — | Ключ Claude API |
| `CLAUDE_MODEL` | `claude-opus-4-8` | Модель для вопросов и рецептов |
| `CLAUDE_PARSER_MODEL` | = `CLAUDE_MODEL` | Модель для разбора покупок (самый частый вызов) |
| `CLAUDE_VISION_MODEL` | = `CLAUDE_MODEL` | Модель для чтения фото чеков |

**YandexGPT**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `YANDEX_API_KEY` | — | API-ключ сервисного аккаунта |
| `YANDEX_FOLDER_ID` | — | Идентификатор каталога Yandex Cloud |
| `YANDEX_MODEL` | `yandexgpt/latest` | Модель для вопросов. Дешевле: `yandexgpt-lite/latest` |
| `YANDEX_PARSER_MODEL` | = `YANDEX_MODEL` | Модель для разбора покупок |
| `YANDEX_VISION_MODEL` | = `YANDEX_MODEL` | Модель, разбирающая распознанный текст чека |
| `YANDEX_OCR_MODEL` | `page` | Модель Vision OCR: `page` или `handwritten` |

**GigaChat**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `GIGACHAT_AUTH_KEY` | — | Ключ авторизации из личного кабинета (base64) |
| `GIGACHAT_SCOPE` | `GIGACHAT_API_PERS` | Ещё: `GIGACHAT_API_B2B`, `GIGACHAT_API_CORP` |
| `GIGACHAT_MODEL` | `GigaChat-2-Max` | Ещё: `GigaChat-2-Pro`, `GigaChat-2` (дешевле всего) |
| `GIGACHAT_PARSER_MODEL` | = `GIGACHAT_MODEL` | Модель для разбора покупок |
| `GIGACHAT_VISION_MODEL` | = `GIGACHAT_MODEL` | Модель для чеков. Lite картинок не видит |
| `GIGACHAT_CA_BUNDLE` | пусто | Путь к корневому сертификату НУЦ Минцифры |
| `GIGACHAT_VERIFY_SSL` | `true` | `false` отключает проверку TLS — только если иначе никак |

**Цены и отчёты**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `USD_RATE` | `0` | Курс доллара: сводит Claude и российских провайдеров в одну валюту |
| `CURRENCY` | `₽` | Символ валюты в отчётах о **покупках** (не о расходах на API) |
| `YANDEX_PRICE_LITE` / `YANDEX_PRICE_PRO` | `200` / `1200` | Рублей за миллион токенов |
| `GIGACHAT_PRICE_LITE` / `_PRO` / `_MAX` | `200` / `1500` / `1950` | Рублей за миллион токенов |
| `YANDEX_OCR_PRICE_PER_PAGE` | `0.26` | Рублей за распознанную страницу |

**Поведение**

| Переменная | По умолчанию | Описание |
|---|---|---|
| `TIMEZONE` | `Europe/Moscow` | Часовой пояс — от него зависят «сегодня» и границы месяца |
| `FRIDGE_WINDOW_DAYS` | `10` | Сколько дней продукт считается лежащим в холодильнике |
| `CONTEXT_PURCHASES_LIMIT` | `150` | Сколько последних покупок отдавать модели как контекст |
| `CONFIRM_MODE` | `reply` | Подтверждение записи: `reply` (текстом), `reaction` (👍), `quiet` (молча) |
| `DB_PATH` | `data/bot.sqlite3` | Файл базы |
| `LOG_LEVEL` | `INFO` | `DEBUG`, `INFO`, `WARNING` |

Пустое значение (`DB_PATH=`) равносильно отсутствию переменной — подставится
значение по умолчанию. Так что необязательные поля в `.env` можно спокойно
оставлять пустыми.

Если бот слишком «шумит» в чате подтверждениями — поставьте `CONFIRM_MODE=reaction`.

## Сколько это стоит

Платятся только вызовы нейросети — Telegram и SQLite бесплатны. Сообщения без
цифр в API не уходят совсем. Три вида вызовов:

| Что | Порядок величин | Какая модель |
|---|---|---|
| Разбор текста о покупке | ≈ 500–900 токенов входа, 100–300 выхода | `*_PARSER_MODEL` |
| **Фото чека** | ≈ 1500–5000 токенов входа (картинка) + разбор позиций | `*_VISION_MODEL` |
| Вопрос / рецепт | зависит от `CONTEXT_PURCHASES_LIMIT` | `*_MODEL` |

Фото стоит заметно дороже текста: картинка 2576 px — это несколько тысяч токенов.
Но чеки приходят единицами в неделю, а не десятками в день, так что в общем
счёте это немного. У YandexGPT чек считается иначе: распознавание тарифицируется
за страницу, а модель получает уже текст — обычно это дешевле картинки.

Порядок величин для семьи из двух человек — единицы долларов (или несколько
сотен рублей) в месяц на старших моделях. Точные цены — на страницах тарифов:
[Anthropic](https://platform.claude.com/docs/en/pricing),
[Yandex Cloud](https://yandex.cloud/ru/docs/foundation-models/pricing),
[GigaChat](https://developers.sber.ru/docs/ru/gigachat/api/tariffs).

### Считайте по факту, а не по прикидкам

Бот ведёт собственный учёт: после каждого обращения к API он берёт из ответа
`usage` (сколько токенов реально ушло), считает стоимость по прайсу из `usage.py`
и складывает в таблицу `usage_log` вместе с именем провайдера и валютой.
Команда `/cost` показывает результат:

```
Расходы на нейросеть за июль
Всего: 214 ₽ + $1.42 за 350 вызовов
≈ 342 ₽ по курсу 90 ₽/$

По провайдерам
• GigaChat — 214 ₽ (330)
• Claude — $1.42 (20)

По операциям
• чтение чеков — $1.42 (20)
• разбор покупок — 186 ₽ (300)
• вопросы — 28 ₽ (30)

Токенов: 582K вход / 128K выход
При таком темпе за месяц выйдет ~331 ₽ + $2.20
```

Это и есть ответ на вопрос «какой дешевле»: раздел **По провайдерам** появляется,
как только в периоде оказались расходы больше чем одного.

Периоды те же, что у `/stats`: `/cost неделя`, `/cost сегодня`, `/cost всё`.
Прогноз на месяц показывается только когда смотрите текущий месяц целиком.

**Про валюты.** Anthropic списывает доллары, Яндекс и Сбер — рубли. Складывать
их бот не станет: в отчёте они идут через `+`. Задайте примерный курс в
`USD_RATE`, и появится сводная строка `≈ N ₽`; без курса бот об этом напомнит.

**Про точность прайса.** Цены Anthropic зашиты в `usage.py` и там же датированы.
У Яндекса и Сбера тариф зависит от договора и объёма, поэтому в `.env` стоят
ориентировочные значения — **впишите свои реальные ставки** (`YANDEX_PRICE_*`,
`GIGACHAT_PRICE_*`), иначе сравнение будет по чужому прайсу.

Стоимость считается **в момент вызова и хранится посчитанной**: цены со временем
меняются, и пересчитывать прошлое по новому прайсу было бы неверно.

**Как снизить расходы:**

1. **Разбор переключить на дешёвую модель.** Задача простая и структурированная,
   старшая модель тут не нужна:
   ```ini
   CLAUDE_PARSER_MODEL=claude-haiku-4-5
   YANDEX_PARSER_MODEL=yandexgpt-lite/latest
   GIGACHAT_PARSER_MODEL=GigaChat-2
   ```
   Основную модель при этом оставьте старшей — рецепты и ответы на вопросы
   заметно выигрывают от умной модели.
2. **Писать покупки одним сообщением**, а не пятью подряд — один вызов вместо пяти.
3. **Сравните провайдеров на своих же данных** — для этого всё и сделано.
   Неделя на одном, неделя на другом, `/cost всё` в конце.
4. Следите за расходом в консоли провайдера, там же обычно можно поставить лимит.

```
family-budget-bot/
├── bot.py                 точка входа: запуск, middleware, меню команд
├── config.py              чтение .env
├── models.py              категории + схема разбора (JSON Schema)
├── prompts.py             все системные промпты
├── llm.py                 фасад: что просим у нейросети, независимо от провайдера
├── providers/
│   ├── base.py            общий интерфейс провайдера
│   ├── claude.py          Claude API (Anthropic)
│   ├── yandex.py          YandexGPT + Yandex Vision OCR для чеков
│   └── gigachat.py        GigaChat: токен, вызов функции, загрузка картинок
├── usage.py               прайсы провайдеров и подсчёт стоимости вызова
├── images.py              подготовка фото чека: поворот, размер, формат
├── db.py                  SQLite: схема и запросы
├── services.py            периоды, форматирование, отчёты, контекст для модели
├── handlers/
│   ├── commands.py        /stats, /fridge, /recipe, /ask, /cost, /provider …
│   └── messages.py        текст и фотографии: вопрос это, покупка или чек
├── data/                  база (в .gitignore)
├── requirements.txt
└── .env.example
```

Что где менять:

* **не те категории** → `models.py`, список `Category`;
* **бот неверно разбирает покупки** → `prompts.py`, `PARSER_SYSTEM`;
* **бот неверно читает чеки** → `prompts.py`, `RECEIPT_SYSTEM` (или
  `RECEIPT_TEXT_SYSTEM`, если чеки читает YandexGPT через OCR);
* **не нравится тон ответов** → `prompts.py`, `ASSISTANT_SYSTEM` / `RECIPE_SYSTEM`;
* **неверный прайс** → `.env` (`YANDEX_PRICE_*`, `GIGACHAT_PRICE_*`) или `usage.py`;
* **вид отчётов** → `services.py`;
* **новая команда** → `handlers/commands.py`;
* **четвёртый провайдер** → новый файл в `providers/` + строчка в `providers/__init__.py`.

### Как добавить провайдера

Наследуйтесь от `providers.base.Provider` и реализуйте `complete` (свободный
текст) и `complete_json` (ответ по схеме `PARSED_MESSAGE_SCHEMA_FLAT`). Для чеков
— либо `read_image`, либо `ocr`, в зависимости от того, что умеет провайдер;
соответствующий флаг `supports_images` / `supports_ocr` подскажет `llm.py`,
каким путём идти. Ошибки, которые чинятся настройками или деньгами, бросайте
как `LLMError` — бот покажет их текст прямо в чате.

## Приватность и безопасность

* `.env` и папка `data/` — в `.gitignore`. **Перед первым `git push` проверьте,
  что там нет вашего токена:** `git status` и `git log -p | grep -i sk-ant`.
* **Куда уходят данные — зависит от `LLM_PROVIDER`.** Тексты сообщений с цифрами
  (для разбора), выжимка из базы (для ответов) и фотографии чеков отправляются
  выбранному провайдеру: Anthropic в США, Yandex Cloud или Сберу в России.
  Переключая провайдера, вы меняете и того, кто это видит. Переписка целиком
  не отправляется никому — только то, что бот распознал как покупку.
* Запросам в Yandex Vision OCR бот проставляет `x-data-logging-enabled: false` —
  чеки не должны оседать в логах Яндекса.
* База лежит локально в `data/bot.sqlite3`. Это обычный SQLite-файл без шифрования —
  на сервере позаботьтесь о правах доступа (`chmod 600`).
* `ALLOWED_CHAT_IDS` — единственное, что мешает постороннему добавить вашего бота
  к себе и тратить ваш баланс API. Заполните его.
* `GIGACHAT_VERIFY_SSL=false` отключает проверку сертификата: соединение
  шифруется, но никто не подтверждает, что на том конце действительно Сбер.
  Правильный путь — поставить сертификат и указать `GIGACHAT_CA_BUNDLE`.
* Если токен утёк: @BotFather → `/mybots` → **API Token** → **Revoke**.
  Ключ провайдера — удалите в его консоли и создайте новый.

## Если что-то не работает

**Бот молчит на обычные сообщения, но отвечает на команды.**
Не выключен privacy mode. См. [шаг 1](#1-создать-бота-в-telegram) — и не забудьте
после этого удалить бота из группы и добавить заново.

**Бот вообще не отвечает в группе.**
Проверьте лог. Если там `Сообщение из чужого чата -100…` — этот ID не вписан в
`ALLOWED_CHAT_IDS`.

**`Не задана обязательная переменная …`**
Нет файла `.env` или в нём пустое значение. `cp .env.example .env` и заполните.

**`LLM_PROVIDER='…' — допустимые значения: claude, yandexgpt, gigachat`.**
Опечатка в названии провайдера. Других вариантов пока нет.

**`Claude не принял ключ`.**
Ключ неверный или на балансе Anthropic нет денег (Billing в консоли).

**`Yandex Cloud не принял ключ`.**
Проверьте `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`, а главное — роли сервисного
аккаунта: нужна `ai.languageModels.user`, а для чтения чеков ещё `ai.vision.user`.

**`GigaChat не принял ключ`.**
`GIGACHAT_AUTH_KEY` — это длинная строка base64 целиком, а не идентификатор
клиента. Проверьте также `GIGACHAT_SCOPE`: у физлиц `GIGACHAT_API_PERS`.

**`TLS-сертификат GigaChat не проверился`.**
Нет корневого сертификата НУЦ Минцифры. Скачайте с
[gosuslugi.ru/crt](https://www.gosuslugi.ru/crt), укажите путь к `.pem` в
`GIGACHAT_CA_BUNDLE`. Крайняя мера — `GIGACHAT_VERIFY_SSL=false`.

**`Задайте USD_RATE в .env — сведу провайдеров в одну валюту`** в `/cost`.
В периоде есть расходы и в долларах, и в рублях. Впишите примерный курс в
`USD_RATE`, и появится сводная строка.

**`… не умеет читать чеки`.**
В `VISION_PROVIDER` стоит провайдер без зрения. Поставьте `claude` или
`gigachat` (либо `yandexgpt` — он читает через OCR), или выключите
`READ_RECEIPTS`.

**`На балансе Anthropic закончились деньги`.**
Ключ рабочий, но счёт пуст. [console.anthropic.com](https://console.anthropic.com/)
→ **Plans & Billing** → пополнить. Пока баланс нулевой, бот не сможет ни
записывать покупки, ни отвечать на вопросы — он предупредит об этом в чате
(не чаще раза в 10 минут, чтобы не спамить).

**Бот молчит на фото чека.**
Проверьте `READ_RECEIPTS` в `.env` — должно быть `true`. Если в логе
`Не смог скачать файл` — картинка больше 20 МБ, Telegram Bot API такие не отдаёт;
отправьте обычным фото, а не файлом.

**В чеке распознались не все позиции / не сошлось ИТОГО.**
Бот сам предупредит о расхождении. `/undo` удалит запись целиком, дальше либо
переснимите чек ровнее и при хорошем свете, либо впишите позиции текстом.
Если чеки одного магазина стабильно читаются плохо — добавьте пример его формата
в `RECEIPT_SYSTEM` в `prompts.py`.

**Бот записал покупку неправильно.**
`/undo` удалит последнюю запись, после чего напишите её понятнее. Если ошибается
систематически — поправьте `PARSER_SYSTEM` в `prompts.py`.

**Бот принял за покупку то, что покупкой не было.**
Тоже `/undo`. Такое бывает на сообщениях вида «отдал 500 Пете» — добавьте
исключение в промпт.

**`TelegramConflictError: terminated by other getUpdates request`.**
Запущено два экземпляра бота с одним токеном. Остановите лишний.

### v3 — вынос на сервер
Сейчас бот работает через long polling — это значит, что он живёт ровно столько,
сколько открыт терминал на ноутбуке. Для круглосуточной работы нужен сервер.

**Шаг 1. Арендовать VPS.** Хватит самого дешёвого: 1 CPU / 1 GB RAM
(Timeweb, Selectel, Hetzner — от ~200 ₽/мес). ОС — Ubuntu 24.04 LTS.

**Шаг 2. Развернуть.**
```bash
ssh root@ваш-сервер
apt update && apt install -y python3-venv git
git clone https://github.com/ВАШ_ЛОГИН/family-budget-bot.git /opt/family-budget-bot
cd /opt/family-budget-bot
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
cp .env.example .env && nano .env      # вписать токены
chmod 600 .env
```

**Шаг 3. Сделать systemd-сервис**, чтобы бот стартовал сам и переживал
перезагрузки — `/etc/systemd/system/family-budget-bot.service`:
```ini
[Unit]
Description=Family Budget Bot
After=network-online.target

[Service]
WorkingDirectory=/opt/family-budget-bot
ExecStart=/opt/family-budget-bot/.venv/bin/python bot.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```
```bash
systemctl daemon-reload
systemctl enable --now family-budget-bot
journalctl -u family-budget-bot -f     # смотреть логи
```

**Шаг 4. Бэкапы базы.** Cron раз в сутки:
```
0 4 * * * sqlite3 /opt/family-budget-bot/data/bot.sqlite3 ".backup /root/backup/bot-$(date +\%F).sqlite3"
```
